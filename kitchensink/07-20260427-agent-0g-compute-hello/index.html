kitchensink/07 — Talk to an agent that runs on 0G Compute

No local model. No OpenAI API key. The LLM runs on a provider's GPU somewhere on the 0G Compute Network.

What this proves. You can run an "open agent" without OpenAI/Anthropic and without a powerful machine. You send POST /api/chat with a message; the server forwards to a 0G Compute provider (selected at setup time) using the broker SDK; the provider's GPU runs inference; the reply comes back.

What pays for it. A throwaway wallet you funded with free testnet OG tokens via the faucet. Every chat turn debits a tiny amount from your broker ledger. If the ledger empties, run npm run setup to top up.

What's NOT here. No streaming, no memory across page reloads, no tools, no agent loop. This is "agent" in the most basic sense — one turn, one reply. Everything richer (multi-turn, tool calls, AXL transport, ENS identity) sits on top of this primitive.
