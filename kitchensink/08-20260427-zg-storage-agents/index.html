kitchensink/08 — Agents as immutable bundles on 0G Storage

Phase 1: an agent is two markdown files (AGENTS.md + SOUL.md) wrapped in a manifest. The manifest's hash is the agent's stable identity. Anyone with the hash can load the agent; talking to it dispatches to 0G Compute.

Why two files? Following agentcompanies.io spec: AGENTS.md defines what the agent does (role, instructions, skills it can use). SOUL.md defines who the agent is (voice, values, personality). They're stored as separate immutable artifacts on 0G Storage and bundled by a tiny manifest.json whose hash is the agent's permanent address.

What you can do here: (1) Load templates → (2) edit the two markdowns → (3) Save to 0G — get back a manifest hash. Then (4) paste that hash into the loader to fetch the agent and (5) chat with it. The agent's responses are generated by 0G Compute, prompted by what you wrote in the two files.

Why immutable matters: the same hash always resolves to the same bytes. Updating the agent means saving a new bundle, getting a new hash. Old hash = old version, forever recoverable. Versioning by hash, like Git.
