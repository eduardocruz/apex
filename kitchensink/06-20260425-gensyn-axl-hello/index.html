kitchensink/06 — Gensyn AXL: two nodes talking

Sponsor: Gensyn (ETHGlobal Open Agents). Bounty: meaningful AXL integration with cross-node message exchange.

What is AXL? Gensyn's Agent eXchange Layer — a single Go binary that gives any application an encrypted, decentralized communication layer over a P2P mesh (built on Yggdrasil + gVisor). No central broker, no server to point at.

What this page demonstrates. Two AXL nodes are running on this machine: node A (listener on tls://127.0.0.1:9001, HTTP API :9002) and node B (peers to A, HTTP API :9012). They have separate ed25519 identities and exchange messages over their own gVisor TCP fabric — not in-process, not via this page's HTTP server (which is just a UI proxy).

How to drive it. Click Refresh on each node to see its public key and peer list. Then send a message from A→B (or B→A) and click Receive on the destination — the body comes back with the sender's X-From-Peer-Id, proving the cross-node hop happened.

What "demonstrating cross-node communication" means for the bounty
The Gensyn prize requires that AXL is the actual message bus, not a substitute for one. This page proves it: when you click Send, the UI calls POST /send on the source node's HTTP bridge with an X-Destination-Peer-Id. The source node packages the bytes, encrypts them, and routes them through the gVisor TCP layer to the destination node — over the TLS link those nodes established themselves at startup. The receiving node's /recv queue holds the message until you poll it. Nothing about this flow uses a central broker, and the two nodes don't share process memory.
